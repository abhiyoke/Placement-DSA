1.
File System:
Basic Concept: Data ko files mein store karta hai.
Data Management: Manual management, high redundancy aur inconsistency.
Data Access: Slow access due to manual searching.
Security: Limited security, manual protection.
DBMS:
Basic Concept: Systematic data storage in tables.
Data Management: Automatic management, low redundancy aur inconsistency.
Data Access: Fast access using SQL queries.
Security: High-level security with user authentication aur authorization.

2.ER model:
Entity: Real-world object (e.g., Student, Book).
Attributes: Characteristics of an entity (e.g., Name, Age).
Relationships: Connections between entities (e.g., Student borrows Book).
Diagram Components:
Rectangle: Entity.
Ellipse: Attribute.
Diamond: Relationship.
Lines: Connections.
ER Model se hum complex database designs ko easily visualize kar sakte hain aur relationships ko samajh sakte hain.

3.Types of Attributes
Simple Attribute:
Yeh wo attributes hote hain jo further divide nahi ho sakte.
Example: Roll Number, Age.
Composite Attribute:
Yeh attributes further divide ho sakte hain multiple sub-parts mein.
Example: Full Name (First Name aur Last Name), Address (Street, City, Zip Code).
Multivalued Attribute:
Yeh attributes multiple values hold kar sakte hain.
Example: Phone Numbers (ek person ke paas ek se zyada phone numbers ho sakte hain), Email Addresses.
Derived Attribute:
Yeh attributes calculate kiye jaate hain doosre attributes ke basis par.
Example: Age (jo Date of Birth se calculate hoti hai), Total Marks (jo individual marks se calculate hota hai).
Key Attribute:
Yeh attribute entity ko uniquely identify karta hai.
Example: Roll Number for Student, Employee ID for Employee.
